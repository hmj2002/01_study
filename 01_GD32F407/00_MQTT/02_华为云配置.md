## 1. 开通华为云账号

1. 访问 [华为云平台](https://www.huaweicloud.com)
2. 注册开通华为云账号
3. 完成**实名认证**（个人认证即可）

## 2. 开通设备接入 IoTDA

**IoTDA** 是 **IoT Device Access** 的缩写，中文全称为**设备接入服务**。它是华为云提供的一款核心物联网平台服务，主要充当物理世界与数字云端之间的"桥梁"。

### 2.1 点击开通免费单元

1. 访问 [IoTDA 控制台](https://console.huaweicloud.com/iotdm/?region=cn-east-3#/dm-portal/instance)
2. 如果你是用來学习或做小规模测试，直接点击**"开通免费单元"**按钮即可，**无需购买付费实例**

#### 创建免费单元步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 选择**区域** | 建议选择 **"华东-上海一"**（`cn-east-3`） |
| ② | 选择**计费模式** | 选择 **"按需计费"** |
| ③ | 选择**规格配置** | 选择 **"免费单元 S0"** |
| ④ | 填写**实例名称** | 按自己喜欢写名称，如 `免费测试单元` |
| ⑤ | 点击**"立即创建"** | 等待创建完成 |

> **免费单元 S0 规格：**
> - 同时在线设备数上限：**1,000**
> - 每日消息数上限：**10,000**
> - 上下行消息 TPS 峰值：**10 TPS**

3. 创建完成后，进入 **IoTDA 实例** 页面，确认实例状态为**"运行中"**

## 2.2 创建产品

产品是设备的集合，是指某一类具有相同能力或特征的设备的合集。

### 操作步骤

1. 在 IoTDA 控制台左侧菜单，点击 **"产品"**
2. 点击 **"创建产品"** 按钮
3. 填写产品信息：

| 参数 | 填写内容 |
|------|----------|
| **所属资源空间** | 默认（如 `DefaultApp_6a28xtyc`） |
| **产品名称** | 自行填写，如 `小黑生活` |
| **协议类型** | **MQTT** |
| **数据格式** | **JSON** |
| **设备类型选择** | **自定义类型** |
| **设备类型** | 按需填写，如 `智能家居` |

4. 点击 **"确定"**，创建产品

### 上传产品模型

创建产品成功后，系统会提示：

> 1. **定义产品模型** —— 通过定义模型，在平台构建一款设备的抽象模型，使平台理解该款设备支持的功能  
> 2. **添加和调试设备** —— 您可以注册测试设备，或者使用平台提供的模拟器，进行在线调试

#### 操作步骤

1. 点击 **"查看详情"** 进入产品详情页
2. 在 **"模型定义"** 区域，点击 **"上传模型文件"**
3. 上传提供的模型文件 `huawei_model.zip`（也可自己进行自定义模型）
4. 点击 **"确定"** 完成上传

> 💡 **提示**：如果没有定义产品模型，设备上报数据时平台仅直接转发，不做解析。

### 新增自定义属性（可选）

如果需要在已有模型基础上新增属性：

1. 在模型定义页面，点击 **"新增属性"**
2. 填写属性信息，例如：

| 参数 | 示例值 |
|------|--------|
| **属性名称** | `Humidity` |
| **属性描述** | 传感器湿度 |
| **数据类型** | `int`（整数型） |
| **访问权限** | 可读 / 可写 |
| **取值范围** | `0` ~ `100` |
| **步长** | `1` |

3. 点击 **"确定"** 保存

## 2.3 添加设备

### 操作步骤

1. 切换回**产品页面**，点击 **"管理设备"**
2. 进入设备管理页面，点击 **"注册设备"**
3. 选择 **"单设备注册"**，填写注册信息：

| 参数 | 填写内容 |
|------|----------|
| **所属产品** | 选择刚才创建的产品（如 `小黑生活`） |
| **设备标识码** | 自定义唯一标识，如 `dev001` |
| **设备密钥** | 自定义密钥（请牢记） |
| **设备名称** | 自定义名称 |

4. 点击 **"确定"**
5. 设备创建成功后，点击 **"保存并关闭"**

### 查看 MQTT 连接参数

1. 在**所有设备**列表中，找到刚注册的设备，点击 **"详情"**
2. 在设备详情页，点击 **"MQTT 连接参数"**
3. 查看并记录以下参数：

| 参数 | 说明 |
|------|------|
| `clientId` | 客户端 ID |
| `username` | 用户名 |
| `password` | 密码 |
| `serverAddr` | 服务器地址 |
| `port` | 端口号 |
| `protocol` | 协议类型 |

> ⚠️ **端口说明：**
> - **非安全 MQTT 协议**端口为 **`1883`**
> - **安全 MQTT 协议（MQTTS）**端口为 **`8883`**

4. 点击 **"一键复制"** 保存所有连接参数，后续代码中需要使用

## 3. Topic 管理

**Topic 管理**的作用是为设备和云平台之间的不同通信内容，定义好各自专属的"信息通道"。

### 系统预置 Topic

华为云 IoTDA 为每个产品预置了以下 Topic（以产品 `小黑生活` 为例）：

| Topic 路径 | 权限 | 用途说明 |
|-------------|------|----------|
| `$oc/devices/{device_id}/sys/messages/up` | 发布 | 设备向平台上报消息 |
| `$oc/devices/{device_id}/sys/messages/down` | 订阅 | 平台向设备下发消息 |
| `$oc/devices/{device_id}/sys/properties/report` | 发布 | 设备属性上报 |
| `$oc/devices/{device_id}/sys/properties/set` | 订阅 | 平台设置设备属性 |
| `$oc/devices/{device_id}/sys/properties/get` | 订阅 | 平台获取设备属性 |
| `$oc/devices/{device_id}/sys/commands/request_id={request_id}` | 订阅 | 平台下发命令 |
| `$oc/devices/{device_id}/sys/commands/response/request_id={request_id}` | 发布 | 设备响应平台命令 |

> 💡 **说明**：`{device_id}` 需要替换为实际的设备 ID。
